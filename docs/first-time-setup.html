TraderBot runs two strategies: a Sentiment Trader that buys stocks and options on Public.com based on multi-source news sentiment, and Carpet Bagger that trades Kalshi sports prediction markets — designed to cover the bot's compute costs.

You'll need accounts with:

  • 🏦 Public.com Brokerage for stock and options trading. Enable API access in Account Settings.
  • ☁️ AWS Lambda, EventBridge, DynamoDB, SNS, and Secrets Manager. Free tier covers most usage; expect ~$5/mo.
  • 🤖 Anthropic Claude API for trade decisions and the evening suggestion engine. $5–20/mo depending on usage.
  • 📈 Polygon, Finnhub, MarketAux, NewsAPI, Alpha Vantage Sentiment data sources. All have free tiers sufficient for the bot's usage.
  • 🎲 Kalshi Prediction markets for Carpet Bagger. Optional — skip if you only want the Sentiment Trader.

Infrastructure
AWS Setup
The bot runs as an AWS Lambda function triggered by EventBridge schedules. SNS handles email alerts. Secrets Manager holds all API keys so nothing is hardcoded.

Region where your Lambda, DynamoDB, and Secrets Manager secret live. Not persisted across sessions — always verify before saving.

Your Lambda function can't read a local .env file — it pulls all keys from Secrets Manager at runtime. After saving your .env on the last step, upload it once with:

aws secretsmanager create-secret --name trading-bot/secrets --secret-string file://.env

Leave the default unless you've already created a secret with a different name.

Security
Approval Token
Evening suggestion emails include one-click trade approval links. Each link is signed with HMAC-SHA256 using this secret so only you — with access to your inbox — can approve trades. Links expire after 20 hours.

A random 256-bit secret. Generate one now and save it in AWS Secrets Manager under SUGGESTION_TOKEN_SECRET. If this value ever changes, all outstanding approval links stop working.

Next steps:

  1. Confirm .env is in the root of your trader-bot directory.
  2. Upload your secrets to AWS Secrets Manager:
    aws secretsmanager create-secret --name trading-bot/secrets --secret-string file://.env
  3. Create the DynamoDB log table:
    aws dynamodb create-table --table-name trading-bot-logs --attribute-definitions AttributeName=trade_id,AttributeType=S --key-schema AttributeName=trade_id,KeyType=HASH --billing-mode PAY_PER_REQUEST
  4. Run ./deploy.sh to build the Lambda package and create all EventBridge schedules.
  5. Verify: source .venv/bin/activate && python -c "import requests, anthropic, boto3, dotenv; print('OK')"
  6. When making code changes during the day, use Dev Mode to pause trading instantly.
