Welcome
Let's set up TraderBot
This wizard walks you through every configuration value and generates a .env file ready to drop into your project. Your secrets are never sent anywhere — everything runs in your browser.
TraderBot runs two strategies: a Sentiment Trader that buys stocks and options on Public.com based on multi-source news sentiment, and Carpet Bagger that trades Kalshi sports prediction markets — designed to cover the bot's compute costs.

You'll need accounts with:

  • 🏦 Public.com Brokerage for stock and options trading. Enable API access in your account settings.
  • ☁️ AWS Lambda, EventBridge, DynamoDB, SNS, and Secrets Manager. Free tier covers most usage; expect around $5/mo.
  • 🤖 Anthropic Claude API for trade decisions and the evening suggestion engine. $5–20/mo depending on usage.
  • 📈 Polygon.io Market data. The free tier (5 req/min) works fine — the scanner respects rate limits automatically.
  • 📰 Finnhub, MarketAux, NewsAPI, Alpha Vantage Sentiment data sources. All have free tiers sufficient for the bot's usage.
  • 🎲 Kalshi Prediction markets for Carpet Bagger. Optional — skip if you only want the Sentiment Trader.
Broker
Public.com
The bot buys stocks and options through your Public.com brokerage account using their personal API. Fractional shares are supported — trades start at $2.

Find this in Public.com → Settings → API. Generate a personal API secret if you haven't already. Keep it private — it grants full trading access to your account.

Carpet Bagger
Kalshi (optional)
Carpet Bagger trades Kalshi prediction markets on live sports game outcomes. Its goal is to generate enough to cover the bot's compute costs — a small float of $50–100 is enough. Skip this step if you only want the Sentiment Trader.
The Kalshi RSA private key is sensitive. In production the bot pulls it from AWS Secrets Manager automatically — that's the recommended approach. You can paste it below for local testing only.

Found in Kalshi → Settings → API. Required for Carpet Bagger to scout and trade.

Paste the full PEM-encoded RSA private key associated with your Kalshi API key, including the BEGIN and END lines. In production, store this in AWS Secrets Manager under the key KALSHI_RSA_PRIVATE_KEY.

Security
Approval Token
Evening suggestion emails include one-click trade approval links. Each link is signed with HMAC-SHA256 using this secret so only you — with access to your inbox — can approve trades. Links expire after 20 hours.

A random 256-bit secret. Generate one now and store it safely. If this value ever changes, all outstanding approval links stop working.

Behavior
Feature Toggles
Configure how the bot operates. These can be changed at any time by updating environment variables on your Lambda function — no redeploy needed.
Require sell approval When on, the bot emails you before closing any position rather than auto-selling. Turn this on while you're getting comfortable with the bot's behavior.
Trade debug mode When on, you receive an email for every Claude agent decision — including ones that don't result in a trade. Useful for tuning thresholds in the first week.

Other parameters — watchlist tickers, sentiment thresholds, risk limits, HYSA benchmark rate — are set directly in config/settings.py. The defaults are reasonable starting points. Adjust after your first week once you've seen how the scanner behaves on your watchlist.

Download your .env
Place the downloaded file in the root of the trader-bot directory alongside lambda_function.py. Then follow the next steps below to finish your AWS setup.

Next steps:

  1. Place the .env in the root of your trader-bot directory.
  2. Upload your secrets to AWS Secrets Manager:
    aws secretsmanager create-secret --name trading-bot/secrets --secret-string file://.env
  3. Create the DynamoDB log table:
    aws dynamodb create-table --table-name trading-bot-logs \
      --attribute-definitions AttributeName=trade_id,AttributeType=S \
      --key-schema AttributeName=trade_id,KeyType=HASH \
      --billing-mode PAY_PER_REQUEST
  4. Run ./deploy.sh to build the Lambda package and create all EventBridge schedules.
  5. Test locally: source .venv/bin/activate && python3 test_scan.py
